# Golden Demo — Extract Test Vectors

This command runs automatically after `/speckit.plan`. It scans the feature
specification and plan for acceptance criteria items, then captures them as
test vectors for later use by the drift-check step.

## Steps

1. Locate the current feature directory and read `spec.md`.

2. Read `plan.md` from the same directory.

3. Scan both files for acceptance criteria. Look for:
   - Sections or bullet points that include phrases such as:
     "given", "when", "then", "should", "must", "expect", "returns",
     "acceptance criteria", "AC:", "✓", "- [ ]"
   - Any explicit input/output examples (e.g. `input: X → output: Y`)

4. Create the directory `.spec-kit/golden/` if it does not already exist.

5. Write the discovered items to `.spec-kit/golden/test-vectors.md` using
   this exact format:

```markdown
# Test Vectors
<!-- Auto-generated by golden-demo after_plan hook. Do not edit manually. -->
<!-- Source: spec.md + plan.md -->
<!-- Generated: {ISO timestamp} -->

## Vectors

- VECTOR 001: {exact text of the acceptance criterion as found in the source}
- VECTOR 002: {exact text of the acceptance criterion as found in the source}
...
```

6. If no acceptance criteria are found, write the file with this content:

```markdown
# Test Vectors
<!-- Auto-generated by golden-demo after_plan hook. Do not edit manually. -->
<!-- No acceptance criteria detected in spec.md or plan.md. -->
<!-- Add explicit input/output examples or "given/when/then" blocks to your spec. -->
```

7. Report to the user:
   - How many vectors were captured
   - The path of the written file
   - A one-line note: "Review and adjust vectors before running implement."

> **Note:** This step does not generate or run any code. It only captures
> intent from the spec as human-readable, version-controllable artifacts.
